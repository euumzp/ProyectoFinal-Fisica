# Teoría de Git

## ¿Qué es Git?

Git es un programa que se usa para rastrear los cambios en el código o en cualquier proyecto con archivos.  
Es muy útil ya que permite volver a versiones anteriores si algo se rompe o ver exactamente qué cambió y cuándo.  

También es una herramienta clave cuando trabajan varias personas en el mismo proyecto, ya que evita que se pisen los cambios entre sí.  
Cada persona puede trabajar en su parte y luego todo se junta de forma ordenada.

Git fue creado por Linus Torvalds (el mismo que creó Linux) y es un sistema de control de versiones.  
Básicamente, te permite tener un historial completo de tu proyecto, como si tuvieras “guardados” infinitos de tu trabajo.

---

## ¿Cómo funciona Git?

Cuando se usa Git, se crea algo llamado un repositorio, esta es una carpeta donde se guarda tu proyecto junto con su historial de cambios.  
Se puede usar el repositorio solo en tu computadora (de forma local) o subirlo a internet (de forma remota) para compartirlo con otros usuarios.  
Si se trabaja en equipo (con más personas), cada uno tiene su copia del proyecto y puede subir o bajar los cambios del resto.
Algunos comandos básicos de Git son:

- `git init`: crea un nuevo repositorio.  
- `git add .`: prepara los archivos para guardarlos.  
- `git commit -m "mensaje"`: guarda una versión con una descripción.  
- `git push`: sube los cambios al repositorio remoto.  
- `git pull`: descarga los cambios más recientes.

---

## ¿Qué es GitHub?

**GitHub** es una página web (ahora propiedad de Microsoft) que funciona junto con Git.  
No es Git en sí, sino un lugar donde se pueden guardar tus repositorios online, para tenerlos respaldados y compartirlos más fácilmente.  
Existen otras plataformas parecidas, como GitLab o Bitbucket, pero GitHub es la más conocida y usada.  

Con GitHub se puede:
- Subir tu proyecto y compartirlo con otros.  
- Controlar quién puede ver o editar el código.  
- Revisar versiones anteriores.  
- Trabajar en equipo desde distintos lugares.  

---

## ¿Por qué usar Git?

- Te ayuda a no perder trabajo.  
- Se puede recuperar versiones anteriores si algo sale mal.  
- Facilita el trabajo en grupo.  
- Mantiene el orden y la organización de tus archivos.  

## ¿Cuándo usar Git?

- Estés desarrollando un programa, un informe o cualquier proyecto largo.  
- Trabajes en equipo.  
- Quieras tener el control de los cambios que se hace con el tiempo.  

---

## Conclusión

Git es una herramienta súper práctica, te da control sobre tus proyectos, te salva cuando algo falla y hace que trabajar con otros sea mucho más fácil.  
GitHub, por su parte, es el complemento perfecto: una plataforma que te permite subir tu trabajo, compartirlo y colaborar sin complicaciones.
